### 问题与解决

| 问题 | 根因 | 解决方案 |
|------|------|---------|
| 舵机完全不转 | PCA9685_SetFreq 读-改-写模式中 oldmode 可能为脏值，导致 MODE1 配置错误 | 直接写入已知值 0x10/0xA0，不依赖读回 |
| CH1 无法右转 | CH1 实际为 90° 限制舵机（标签 SG90 但行程受限） | 物理换位：90°舵机做挡板(CH0)，180°舵机做分拣(CH1) |
| 状态机最后一步不执行 | hold_ms=0 时直接返回完成，跳过了角度设置 | 改为先执行角度再判断 hold_ms |
| step 0 角度写两次 | Servo_Execute 和 Servo_Update 都执行了 step 0 | step_angle_sent 初始值设为 1 |

### 当前系统配置

```
CH0（90°舵机）= 入口挡板：0°放行 / 90°阻挡（默认）
CH1（180°舵机）= 末端分拣：0°左转B / 90°居中（默认） / 180°右转C

PWM 参数：SERVO_MIN=110(0.54ms), SERVO_MAX=500(2.44ms)
I2C 频率：100kHz
PCA9685 频率：50Hz（prescale=121）
```

### 分拣序列

```
A 类：CH1让开(0°)─300ms─→ CH0放行(0°)─800ms─→ CH0阻挡(90°)─1s─→ CH1复位(90°)  总耗时2.1s
B 类：CH0放行(0°)─300ms─→ CH0阻挡(90°)─2s─→ CH1左转(0°)─1s─→ CH1复位(90°)      总耗时3.3s
C 类：CH0放行(0°)─300ms─→ CH0阻挡(90°)─2s─→ CH1右转(180°)─1s─→ CH1复位(90°)     总耗时3.3s
```

### 问题与解决

| 问题 | 根因 | 解决方案 |
|------|------|---------|
| YOLOv8 无法识别电子元器件 | COCO 预训练模型只有 80 类日常物品，不含电子元器件 | 训练自定义 YOLO 模型（3 类，mAP50=86.8%） |
| 颜色检测方案不准确 | 元器件颜色与环境光/背景干扰大，HSV 分割不稳定 | 放弃颜色方案，改用 YOLO 目标检测 |
| labelimg 闪退 | PyQt5 版本兼容性问题 | 自建 OpenCV 标注工具 annotate.py |
| 舵机反复执行同一动作 | 摄像头每帧检测到同一物体，READY 后立即又发 CMD | 添加 5s 类别冷却期 + 日志 SKIP 提示 |
| 标注/图片保存路径不一致 | collect_data.py 和 annotate.py 从不同目录运行 | 手动移动文件到 pc/dataset/ 下 |

### 自定义模型训练结果

```
模型:   YOLOv8n 微调 (yolov8n.pt → custom_electronics.pt)
数据:   60 张图片（48 训练 / 12 验证），61 个标注框
类别:   resistor(20) / led(21) / capacitor(20)
训练:   50 epochs, batch=8, CPU, ~8 分钟
指标:   mAP50=86.8%, Precision=83.3%, Recall=76.2%
模型大小: 6.0MB
```

### 当前系统架构

```
PC (YOLOv8 检测) ──UART 115200──→ STM32F103 ──I2C──→ PCA9685 ──→ 舵机

PC 端状态机:  IDLE → send CMD → WAIT_ACK → WAIT_READY → IDLE
              (5s 超时保护)  (5s 同类别冷却防重复)
```
