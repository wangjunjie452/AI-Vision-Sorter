# AI 视觉分拣系统 — 调试记录

> **项目**：AI 视觉分拣系统（STM32 + PCA9685 + SG90 舵机）
> **调试日期**：2026-06-05
> **调试工具**：串口助手 115200-8N1

---

## 1. 问题记录：舵机不转动

### 1.1 现象

上电后系统正常初始化（LED 亮、蜂鸣器响），串口日志输出正常，但 **舵机完全不转动**。

### 1.2 排查过程

在 `servo_ctrl.c` 中添加 I2C 诊断代码，覆盖以下环节：

| 排查项 | 方法 | 结果 |
|--------|------|------|
| PCA9685 是否在线 | `HAL_I2C_IsDeviceReady()` 探测 | ONLINE |
| MODE1 寄存器写入 | 写入 `0x10`(Sleep) → `0xA0`(Restart+AutoInc) | 全部 OK |
| PRESCALE 寄存器写入 | 写入 121（50Hz） | OK |
| 寄存器读回验证 | 读回 MODE1=0xA0, PRESCALE=121 | 与写入一致 |
| PWM 通道写入 | 每次 `Servo_SetAngle` 检查返回值 | 全部 OK |

### 1.3 结论

**软件层面（I2C 通信、PCA9685 配置、PWM 计算）均正常，问题出在硬件接线。**

最可能原因（按概率排序）：

1. **PCA9685 的 V+（舵机电源）未接通** — PCA9685 开发板上 VCC（逻辑电源 3.3V）和 V+（舵机电源 5~6V）是独立的，V+ 不接则 PWM 输出无驱动能力
2. **I2C 数据线松动或接触不良** — SCL/SDA 接线虚焊或杜邦线接触不良
3. **舵机信号线接错通道** — 舵机信号线插在 PCA9685 的非 CH0/CH1 通道上

### 1.4 解决方法

确认硬件接线后，舵机正常转动。

## 2. 舵机默认状态

**两个舵机的默认（归位）状态均为 90°。**

| 舵机 | 通道 | 默认角度 | 物理效果 |
|------|------|---------|---------|
| CH0（入口分拣） | PCA9685 CH0 | 90° | 纸板竖直，阻挡物品（`ANGLE_GATE_BLOCK`） |
| CH1（末端落料） | PCA9685 CH1 | 90° | 纸板居中，挡住物品（`ANGLE_DIVERT_CENTER`） |

上电初始化流程：
```
Servo_Init() → CH0=90°, CH1=90°
校准扫描 CH0: 0° → 90° → 180° → 90°（复位）
校准扫描 CH1: 0° → 90° → 180° → 90°（复位）
校准完成后 → CH0=90°, CH1=90°（默认状态）
```

## 3. 校准扫描日志与视觉观察差异

### 3.1 日志记录

校准扫描阶段，串口日志显示 CH0 和 CH1 各成功执行了 4 次角度设置：

```
=== 开始校准扫描 ===
CAL: CH0 -> 0 deg (down/pass)    ← CH0 第1次：pulse=80  → 0°
  [PWM] CH0 angle=  0 pulse= 80 OK
CAL: CH0 -> 90 deg (block)       ← CH0 第2次：pulse=320 → 90°
  [PWM] CH0 angle= 90 pulse=320 OK
CAL: CH0 -> 180 deg (push/up)    ← CH0 第3次：pulse=560 → 180°
  [PWM] CH0 angle=180 pulse=560 OK
CAL: CH0 -> 90 deg (复位)        ← CH0 第4次：pulse=320 → 90°
  [PWM] CH0 angle= 90 pulse=320 OK
CAL: CH1 -> 0 deg (left)         ← CH1 第1次：pulse=80  → 0°
  [PWM] CH1 angle=  0 pulse= 80 OK
CAL: CH1 -> 90 deg (center)      ← CH1 第2次：pulse=320 → 90°
  [PWM] CH1 angle= 90 pulse=320 OK
CAL: CH1 -> 180 deg (right)      ← CH1 第3次：pulse=560 → 180°
  [PWM] CH1 angle=180 pulse=560 OK
CAL: CH1 -> 90 deg (复位)        ← CH1 第4次：pulse=320 → 90°
  [PWM] CH1 angle= 90 pulse=320 OK
=== 校准扫描完成 ===
```

### 3.2 视觉观察

**CH0 舵机只看到 2 次明显转动**：第1次向左（推料方向），第2次转回 90° 复位。

### 3.3 差异原因分析

这不是软件 bug，而是 **物理角度差太小导致肉眼看不出变化**：

| CH0 动作 | 角度变化 | 物理效果 | 视觉感知 |
|----------|---------|---------|---------|
| 0° → 90° | +90° | 纸板从"放行"位置转到"阻挡"位置 | ⚠️ 不明显（小角度位移） |
| 90° → 180° | +90° | 纸板从"阻挡"位置转到"向上推料"位置 | ✅ 明显（大角度位移） |
| 180° → 90° | -90° | 纸板从"推料"位置转回"阻挡"位置 | ✅ 明显（大角度位移） |

**关键原因**：SG90 舵机的实际机械行程约 0°~180°，但：

- `SERVO_MIN=80`（≈0.98ms）、`SERVO_MAX=560`（≈6.84ms）的 PWM 范围远超标准 SG90 的 0.5ms~2.4ms
- 超出标准范围的脉宽会被舵机内部电路截断到极限位置
- **0° 和 90° 的实际物理位置可能非常接近**（因为 pulse=80 和 pulse=320 都在舵机的有效响应范围内，但 0° 被截断到了某个位置）

### 3.4 建议校准

如果需要 CH0 在 0° 和 90° 之间有更明显的视觉区分，需要调整 PWM 范围：

```c
// servo_ctrl.h 中当前值
#define SERVO_MIN   80     // ≈ 0.98ms
#define SERVO_MAX   560    // ≈ 6.84ms

// 建议校准为 SG90 标准范围（根据实际舵机微调）
#define SERVO_MIN   102    // ≈ 1.25ms (对应 0°)
#define SERVO_MAX   512    // ≈ 2.50ms (对应 180°)
```
