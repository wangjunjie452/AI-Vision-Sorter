## 二、芯片选型详解

### 2.1 主控芯片：STM32F103C8T6

```
┌──────────────────────────────────────────────┐
│           STM32F103C8T6 资源概览              │
│                                              │
│  内核：    ARM Cortex-M3                      │
│  主频：    72 MHz                            │
│  Flash：   64 KB                             │
│  SRAM：    20 KB                             │
│  GPIO：    37 个                             │
│  USART：   3 路（USART1/2/3）                │
│  I2C：     2 路（I2C1/I2C2）                 │
│  SPI：     2 路（SPI1/SPI2）                 │
│  TIM：     4 个通用定时器 + 1 个高级定时器     │
│  ADC：     2 路 12-bit，10 通道              │
│  封装：    LQFP48                            │
│  工作电压：2.0 ~ 3.6V                        │
│  价格：    ~¥8（最小系统板）                  │
└──────────────────────────────────────────────┘
```

**选型理由：**
- 资料极多，遇到问题容易搜到答案
- 价格低，最小系统板 ¥8
- 外设资源足够本项目（1 路 USART + 1 路 I2C + 若干 GPIO）
- HAL 库 + STM32CubeIDE 生态成熟

### 2.2 视觉推理芯片：K210（阶段二）

```
┌──────────────────────────────────────────────┐
│              K210 资源概览                     │
│                                              │
│  内核：    RISC-V 双核 64-bit @ 400MHz       │
│  KPU：     神经网络加速器，0.8 TOPS           │
│  SRAM：    8 MB                              │
│  Flash：   16 MB SPI Flash                   │
│  DVP：     数字视频接口（接摄像头）            │
│  SPI：     4 路                              │
│  UART：    3 路                              │
│  I2C：     3 路                              │
│  FPIOA：   引脚自由映射（任意功能到任意引脚）  │
│  价格：    ~¥28（Maix Bit 开发板）           │
└──────────────────────────────────────────────┘
```

**选型理由：**
- 专用 KPU 硬件加速器，推理性能远超同价位通用 MCU
- MaixPy MicroPython 开发，代码量少，上手快
- MaixHub 在线训练平台支持零代码模型训练
- 社区活跃，YOLO/Face Detection 预训练模型丰富

### 2.3 舵机驱动：PCA9685

```
┌──────────────────────────────────────────────┐
│             PCA9685 资源概览                   │
│                                              │
│  接口：    I2C（地址 0x40，默认）             │
│  通道：    16 路 PWM 输出                    │
│  分辨率：  12-bit（4096 级）                  │
│  PWM频率：可调 24Hz ~ 1526Hz                 │
│  供电：    逻辑 3.3V/5V，舵机独立 V+ 端子    │
│  价格：    ~¥6                               │
│  优势：    占用 STM32 仅 2 个引脚（I2C）      │
│           就能控制最多 16 个舵机              │
└──────────────────────────────────────────────┘
```

**选型理由：**
- 仅占用 STM32 的 2 个 I2C 引脚，扩展 16 路舵机
- 避免直接用 STM32 的 TIM 输出 PWM（F103 定时器通道有限）
- 后续扩展舵机数量无需改硬件，只需接线

### 4.2 帧格式定义

```
┌─────────────────────────────────────────────────────────────────┐
│  指令帧（PC/K210 → STM32）                                       │
│                                                                 │
│  ┌──────┬──────┬───────────┬───────────┬───────────┬─────┐     │
│  │ 帧头  │ 分隔 │  参数1    │  参数2    │  参数3    │ 帧尾│     │
│  │ 4Byte│ 1B   │  变长     │  变长     │  变长     │ 1B  │     │
│  ├──────┼──────┼───────────┼───────────┼───────────┼─────┤     │
│  │ CMD  │  :   │  类别ID   │  X坐标    │  Y坐标    │ \n  │     │
│  └──────┴──────┴───────────┴───────────┴───────────┴─────┘     │
│                                                                 │
│  示例：CMD:0,160,120\n                                          │
│  示例：CMD:1,200,100\n                                          │
│  示例：CMD:-1,0,0\n          // 无目标                          │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  心跳帧                                                         │
│                                                                 │
│  PC/K210 → STM32：  "PING\n"                                    │
│  STM32 → PC/K210：  "PONG\n"                                    │
│  间隔：  每 5 秒发送一次                                        │
│  超时：  10 秒未收到 PING 则 STM32 进入待机模式                  │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  状态帧（STM32 → PC/K210）                                       │
│                                                                 │
│  "ACK:<类别ID>\n"     确认收到并开始执行                         │
│  "READY\n"            动作执行完毕，可接受下一条指令              │
│  "ERR\n"              执行异常                                   │
│  "PONG\n"             心跳响应                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 通信时序

```
  PC/K210                              STM32
     │                                    │
     │  ──── "CMD:0,160,120\n" ──────►   │
     │                                    │  解析指令
     │                                    │  舵机动作
     │  ◄──── "ACK:0\n" ──────────────   │
     │                                    │  执行分拣
     │                                    │  动作完成
     │  ◄──── "READY\n" ─────────────    │
     │                                    │
     │  ──── "CMD:1,200,100\n" ──────►   │
     │                                    │
     │            ... 循环 ...            │
     │                                    │
     │  ──── "PING\n" ────────────────►  │  每 5s
     │  ◄──── "PONG\n" ──────────────    │
     │                                    │
```

### 4.4 指令类别 ID 映射表

```
┌────────┬──────────────┬──────────────────────────────────────────────────────┐
│ 类别ID │  含义         │  STM32 动作序列                                      │
├────────┼──────────────┼──────────────────────────────────────────────────────┤
│  0     │  类型A物体   │  CH1让开(0°) → CH0放行(0°) → CH0阻挡(90°) → CH1复位 │
│  1     │  类型B物体   │  CH0放行(0°) → CH0阻挡(90°) → 等物品到 → CH1左转(0°) → 复位│
│  2     │  类型C物体   │  CH0放行(0°) → CH0阻挡(90°) → 等物品到 → CH1右转(180°) → 复位│
│  -1    │  无目标      │  CH0→90°(阻挡), CH1→90°(居中)                        │
│  -2    │  急停        │  立即停止所有舵机                                     │
└────────┴──────────────┴──────────────────────────────────────────────────────┘

注：类别 ID 可根据实际训练模型自由扩展
```

### 5.2 STM32 端模块架构

```

│                     STM32 固件架构                               │
│                                                                 │
│  ┌────────────────────────────────────────────────────────────┐ │
│  │                    main.c（主循环）                         │ │
│  │                                                            │ │
│  │   main() {                                                 │ │
│  │       HAL_Init();                                          │ │
```

### 5.4 STM32 舵机控制核心逻辑

#### PWM 参数

```c
// PCA9685 @50Hz，周期 20ms，分辨率 4096
// SG90 标准：0.5ms(0°) ~ 2.5ms(180°)
// 计算：0.5/20*4096=102, 2.5/20*4096=512
// 实际校准值（预留余量防卡机械限位）：

#define SERVO_MIN   110    // 0° → 约 0.54ms
#define SERVO_MAX   500    // 180° → 约 2.44ms

void Servo_SetAngle(uint8_t channel, uint16_t angle) {
    if (angle > 180) angle = 180;
    uint16_t pulse = SERVO_MIN + (SERVO_MAX - SERVO_MIN) * angle / 180;
    PCA9685_SetPWM(channel, 0, pulse);
}
```

#### 舵机通道与角度定义

```c
// CH0 入口挡板（90° 舵机，只用 0° 和 90°）
#define ANGLE_GATE_BLOCK   90   // 阻挡（默认，纸板竖直挡住物品）
#define ANGLE_GATE_PASS    0    // 放行（纸板倒下，物品落入 A 箱）

// CH1 末端落料分拣（180° 舵机）
#define ANGLE_DIVERT_LEFT  0    // 左转 → B 箱
#define ANGLE_DIVERT_CENTER 90  // 居中（默认）
#define ANGLE_DIVERT_RIGHT 180  // 右转 → C 箱

// 等待时间
#define DELAY_GATE_OPEN_MS      800   // A 类：CH0 放行后等待物品掉落
#define DELAY_ITEM_ARRIVE_MS    2000  // B/C 类：物品从入口传送到 CH1 的等待时间
```

#### 非阻塞状态机动作序列

采用状态机替代阻塞式 `HAL_Delay`，每步只设置舵机角度并记录时间戳，
主循环轮询 `Servo_Update()` 检查是否到达等待时间，不阻塞串口通信。

```c
// 状态机步骤结构体
typedef struct {
    uint8_t  channel;   // 舵机通道
    uint16_t angle;     // 目标角度
    uint16_t hold_ms;   // 该步骤停留时间（0=执行后立即完成）
} ServoStep_t;
```

**A 类序列（落入 A 箱）：**

```c
static const ServoStep_t seq_class_a[] = {
    { CH1, 0°(让开),  300ms },   // CH1 先转开让路
    { CH0, 0°(放行),  800ms },   // CH0 放行，物品掉入 A 箱
    { CH0, 90°(阻挡), 1000ms },  // CH0 挡回，等物品完全掉落
    { CH1, 90°(复位), 0 },       // CH1 复位，完成
};
// 时序：CH1转开 → CH0放行 → CH0挡回 → CH1复位
// 总耗时：约 2.1s
```

**B 类序列（分拣到 B 箱）：**

```c
static const ServoStep_t seq_class_b[] = {
    { CH0, 0°(放行),  300ms },    // CH0 放行，让物品进入传送带
    { CH0, 90°(阻挡), 2000ms },   // CH0 阻挡，等物品传送到 CH1 位置
    { CH1, 0°(左转),  1000ms },   // CH1 左转，物品落入 B 箱
    { CH1, 90°(复位), 0 },        // CH1 复位，完成
};
// 时序：CH0放行 → CH0阻挡等2s → CH1左转1s → CH1复位
// 总耗时：约 3.3s
```

**C 类序列（分拣到 C 箱）：**

```c
static const ServoStep_t seq_class_c[] = {
    { CH0, 0°(放行),  300ms },    // CH0 放行，让物品进入传送带
    { CH0, 90°(阻挡), 2000ms },   // CH0 阻挡，等物品传送到 CH1 位置
    { CH1, 180°(右转), 1000ms },  // CH1 右转，物品落入 C 箱
    { CH1, 90°(复位), 0 },        // CH1 复位，完成

// 时序：CH0放行 → CH0阻挡等2s → CH1右转1s → CH1复位

```

#### 状态机流程

```
Servo_Execute(cmd)
  ├─ 根据 classId 选择动作序列
  ├─ 执行 step[0]：Servo_SetAngle(channel, angle)
  ├─ 记录 step_start_tick
  └─ 返回（非阻塞）

主循环 {
  Servo_Update()
    ├─ step_angle_sent=0？ → 发送角度，记录 tick
    ├─ hold_ms=0？ → 完成，发 READY
    ├─ elapsed < hold_ms？ → 等待中
    └─ 时间到 → step_idx++，继续下一步
}
```

#### 关键修复记录

| 问题 | 修复 |
|------|------|
| step 0 角度被重复写入 | `step_angle_sent` 初始值设为 1，避免 `Servo_Update` 重复执行 |
| 最后一步 hold_ms=0 不执行角度 | 状态机先执行角度再判断 hold_ms，确保最后一步的角度命令被发送 |
| PCA9685 初始化读-改-写导致脏值 | 直接写入已知值 `0x10`/`0xA0`，不依赖读回 |
| SERVO_MAX=560 超出 SG90 行程 | 校准为 500（2.44ms），在 SG90 标准范围内 |

## 十一、调试记录与经验教训

> **更新日期**：2026-06-05
> **问题级别**：🔴 严重（舵机完全不转）

### 11.1 问题描述

上电后系统初始化正常（LED 亮、蜂鸣器响、串口通信正常），但 **PCA9685 驱动的两个舵机完全不转动**。I2C 总线上的 PCA9685 芯片地址（0x40）可被探测到，但 PWM 输出无任何响应。

### 11.2 根因分析

**根本原因在 `PCA9685_SetFreq()` 函数中的"读-改-写"模式存在缺陷。**

#### 问题代码（修改前）

```c
static void PCA9685_SetFreq(float freq)
{
    float prescale = 25000000.0f / (4096.0f * freq) - 1.0f;
    uint8_t oldmode;                                              // ⚠ 未初始化
    uint8_t reg = PCA9685_MODE1;

    HAL_I2C_Master_Transmit(&hi2c1, addr, &reg, 1, 100);         // ⚠ 不检查返回值
    HAL_I2C_Master_Receive(&hi2c1, addr, &oldmode, 1, 100);      // ⚠ 不检查返回值

    PCA9685_WriteReg(PCA9685_MODE1, (oldmode & 0x7F) | 0x10);    // 进入 Sleep
    PCA9685_WriteReg(PCA9685_PRESCALE, (uint8_t)prescale);        // 设置分频
    PCA9685_WriteReg(PCA9685_MODE1, oldmode);                     // ❌ 关键缺陷
    HAL_Delay(5);
    PCA9685_WriteReg(PCA9685_MODE1, oldmode | 0xA0);              // ❌ 关键缺陷
}
```

#### 缺陷叠加分析

```
正常假设：  读回 oldmode = 0x00（PCA9685 上电默认值）
            第3步写入 0x00 → 清除 SLEEP 位 → 振荡器启动 ✅
            第4步写入 0x00 | 0xA0 = 0xA0 → Restart + AutoInc ✅

实际问题 1：I2C 读取失败时（返回值未检查），oldmode 保持栈上随机垃圾值
            例：oldmode = 0x37（栈残留）
            第3步写入 0x37 → 含 SLEEP=1, RESTART=1 等脏位 → 芯片状态混乱 ❌
            第4步写入 0x37 | 0xA0 = 0xB7 → 未知行为 ❌

实际问题 2：热复位（快速重新烧录）后，PCA9685 可能保留上次运行的 MODE1 值
            例：oldmode = 0xA0（上次写入残留）
            第3步写入 0xA0 → RESTART 位仍置位 → 芯片可能不响应 ❌
            第4步写入 0xA0 | 0xA0 = 0xA0 → 看似正确但芯片内部状态不对 ❌
```

**核心问题**：依赖一个可能不可靠的读取值来配置关键寄存器，且无返回值校验。

#### PCA9685 MODE1 寄存器参考

```
┌─────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐
│  7  │  6   │  5   │  4   │  3   │  2   │  1   │  0   │
├─────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│RESTART│EXTCLK│ AI   │ SLEEP│SUB1 │SUB2  │SUB3  │ALLCALL│
└─────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘

关键位：
  Bit 4 (SLEEP)  = 1: 休眠模式，振荡器关闭，PWM 输出无效 ← 舵机不转的直接原因
  Bit 4 (SLEEP)  = 0: 正常工作，振荡器运行
  Bit 7 (RESTART)= 1: 需要重启 PWM（写 1 清除）
  Bit 5 (AI)     = 1: 寄存器地址自动递增（多字节写入需要）
```

### 11.3 修复方案

**改为直接写入已知正确值，不依赖读取结果：**

```c
static uint8_t PCA9685_SetFreq(float freq)
{
    float prescale = 25000000.0f / (4096.0f * freq) - 1.0f;

    PCA9685_WriteReg(PCA9685_MODE1, 0x10);    // ✅ 强制 Sleep（关闭振荡器）
    PCA9685_WriteReg(PCA9685_PRESCALE, 121);   // ✅ 50Hz 分频（直接写已知值）
    HAL_Delay(5);
    PCA9685_WriteReg(PCA9685_MODE1, 0xA0);    // ✅ Restart + AutoIncrement

    // 读回验证（可选，仅用于调试）
    uint8_t mode1 = 0;
    PCA9685_ReadReg(PCA9685_MODE1, &mode1);
    return (mode1 == 0xA0) ? 1 : 0;           // ✅ 校验配置是否生效
}
```

#### 修复要点

| 改动 | 说明 |
|------|------|
| `oldmode` 变量移除 | 不再读取可能脏的寄存器值 |
| 直接写 `0x10` | 无论之前什么状态，强制进入 Sleep |
| 直接写 `0xA0` | 无论之前什么状态，强制 Restart + AutoInc |
| 所有 I2C 操作检查返回值 | 通信失败时能立即发现 |
| 读回验证 | 确认写入的值确实生效 |

### 11.4 PCA9685 初始化正确流程

```
┌─────────────────────────────────────────────────────────────┐
│              PCA9685 初始化时序（50Hz 舵机驱动）               │
│                                                             │
│  1. 探测芯片                                                │
│     HAL_I2C_IsDeviceReady(addr=0x40)  → 确认 I2C 通信正常   │
│                                                             │
│  2. 进入 Sleep（关闭振荡器，才能修改 PRESCALE）               │
│     写 MODE1 = 0x10          (SLEEP=1, 其余=0)              │
│                                                             │
│  3. 设置 PWM 频率                                           │
│     写 PRESCALE = 121        (25MHz / (4096 × 50) - 1)     │
│                                                             │
│  4. 等待振荡器稳定                                           │
│     HAL_Delay(5)              (datasheet 要求 ≥ 500μs)      │
│                                                             │
│  5. 唤醒 + 使能自动递增                                      │
│     写 MODE1 = 0xA0          (RESTART=1, AI=1)              │
│                                                             │
│  6. 读回验证                                                │
│     读 MODE1 → 应为 0xA0                                    │
│     读 PRESCALE → 应为 121                                  │
│                                                             │
│  7. 设置 PWM 占空比                                          │
│     写 LEDn_ON_L/H, LEDn_OFF_L/H → 舵机开始动作             │
└─────────────────────────────────────────────────────────────┘
```

### 11.5 经验总结

```
┌──────────────────────────────────────────────────────────────────┐
│                      调试经验教训                                  │
│                                                                  │
│  1. I2C 外设寄存器操作必须检查返回值                               │
│     └── 不检查 = 静默失败，问题极难定位                           │
│                                                                  │
│  2. 不要用"读-改-写"模式初始化芯片，除非必要                       │
│     └── 上电初始化应直接写入已知正确值，避免残留状态干扰           │
│                                                                  │
│  3. 局部变量未初始化 + I2C 读取失败 = 灾难                         │
│     └── oldmode 在栈上是随机值，读取失败后该随机值被当作配置写入   │
│                                                                  │
│  4. 调试手段：串口日志 + 寄存器读回验证                            │
│     └── 每步 I2C 操作打印返回值，关键寄存器写后读回验证           │
│                                                                  │
│  5. PCA9685 特有注意事项                                          │
│     └── 必须先进 Sleep 才能改 PRESCALE（datasheet 硬性要求）      │
│     └── 改完 PRESCALE 后必须 Restart 才生效                       │
│     └── V+（舵机电源）和 VCC（逻辑电源）是独立供电                │
└──────────────────────────────────────────────────────────────────┘
```

### 11.6 舵机默认状态确认

上电初始化及校准扫描完成后，两个舵机均回到 **90° 默认位置**：

| 舵机 | 通道 | 默认角度 | PWM 脉宽 | 物理效果 |
|------|------|---------|---------|---------|
| CH0 入口分拣 | PCA9685 CH0 | 90° | pulse=320（≈3.9ms） | 纸板竖直，阻挡物品 |
| CH1 末端落料 | PCA9685 CH1 | 90° | pulse=320（≈3.9ms） | 纸板居中，挡住物品 |

校准扫描顺序（上电后自动执行）：
```
CH0: 0° → 90° → 180° → 90°（复位）   每步间隔 2 秒
CH1: 0° → 90° → 180° → 90°（复位）   每步间隔 2 秒
```
