## 一、项目章程

### 1.1 项目背景

嵌入式 AI（边缘智能）是当前硬件工程师岗位的核心竞争力之一。本项目通过从零搭建一套"摄像头 + AI 推理 + 嵌入式控制"的完整系统，积累视觉检测、串口通信、舵机控制、模型部署等全栈能力，形成可展示、可面试、可复刻的个人作品。

### 1.2 项目目标（SMART 原则）

| 维度 | 目标 | 衡量标准 |
|------|------|---------|
| **S**pecific（具体） | 搭建一套 AI 视觉分拣系统：摄像头采集 → YOLO 目标检测 → STM32 控制舵机完成分拣 | 系统端到端跑通 |
| **M**easurable（可量化） | 分拣准确率 ≥ 70%，响应延迟 ≤ 500ms，连续运行 ≥ 30 分钟无崩溃 | 实测数据 |
| **A**chievable（可达成） | 分两阶段：PC 端原型 → 手机独立运行（可升级 K210） | 每阶段有独立可交付物 |
| **R**elevant（相关性） | 覆盖嵌入式开发 + AI 部署 + 系统集成三大能力，直接提升简历竞争力 | 面试可讲完整技术栈 |
| **T**ime-bound（有时限） | 总工期 21 天（2026-05-30 ~ 2026-06-19） | 按里程碑验收 |

### 1.3 项目范围

#### 在范围内（In Scope）

- PC 端 YOLO 目标检测程序（Python）
- STM32 串口通信 + 舵机驱动固件（C/HAL）
- PC 端 GUI 控制面板
- K210 端侧 YOLO 推理部署
- 自定义目标检测模型训练
- 系统联调与测试
- 项目文档与演示视频

#### 不在范围内（Out of Scope）

- 机械臂结构件设计（用现成舵机 + 简单支架替代）
- PCB 量产设计（面包板 / 洞洞板即可）
- 多机协同 / 云端管理
- 商业化产品设计

### 双舵机传送带分拣方案定义（Day 3 更新）

```
CH0（入口分拣 — 纸板阻挡/推料）：
  0°  = 阻挡（默认，挡住物品）
  70° = 放行（让物品通过传送带）
  120°= 推料（纸板推出，把 A 类物品推离传送带）

CH1（末端落料 — 纸板格挡）：
  0°  = 向左打满（格挡转出，物品掉入 B 箱）
  90° = 居中（默认，纸板挡住物品）
  180°= 向右打满（格挡转出，物品掉入 C 箱）

分拣动作序列：
  CMD:0 (A类) → 入口推料：等1s → CH0 120°推料 → CH0 0°复位
  CMD:1 (B类) → 末端左分拣：CH0 70°放行 → CH0 0°阻挡 → 等2s → CH1 0°左打满 → CH1 90°复位
  CMD:2 (C类) → 末端右分拣：CH0 70°放行 → CH0 0°阻挡 → 等2s → CH1 180°右打满 → CH1 90°复位
```

**类别映射（detect.py）：**
```
A类（入口推料）：瓶子、杯子
B类（末端左分拣）：手机、遥控器
C类（末端右分拣）：苹果、香蕉
```

### STM32 引脚分配总表（Day 1 确定）

| 引脚 | 功能 | 模块 | 说明 |

### 供电方案（Day 2 调试确认）

舵机电源必须独立于 PCA9685，不从 V+ 端子取电（Adafruit 板 V+ 未连通到排针）：

```
外部 5V 电源 ──→ 舵机红线（V+）
                 舵机棕线(GND) ──→ 共地 ──→ STM32 GND + PCA9685 GND

ST-Link 5V   ──→ PCA9685 VCC（芯片逻辑供电）
ST-Link 3.3V ──→ STM32 3.3V
ST-Link GND  ──→ 共地

PCA9685 仅负责信号输出（CH0/CH1 → 舵机黄线）
```

8. 串口通信丢包怎么处理？（答：超时重传 + 心跳检测）
